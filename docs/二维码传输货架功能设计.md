# 二维码传输货架功能设计

## 背景

用户在 PC 端管理货架数据，需要一种便捷方式将货架信息传输到手机端。通过二维码扫描，手机可以快速接收货架数据并导入本地数据库。

## 方案概述

**纯离线方案**：PC 端生成二维码（编码货架 JSON 数据），手机端扫码解析并导入。

### 技术约束

| 项目 | 说明 |
|------|------|
| 二维码容量 | 字节模式约 2953 字节 |
| 一个货架（32零件） | JSON 约 4-5KB，pako 压缩后约 2-2.5KB |
| 本次范围 | 单货架传输（最多 32 零件） |

### 需求确认

1. **二维码样式**：✅ 支持自定义 Logo 和颜色
2. **数据量上限**：✅ 单货架（32 零件），压缩后约 2.5KB，可放入二维码
3. **导入策略**：✅ 重名货架处理方式可供用户选择（类似现有 Excel 导入功能）

---

## 功能设计

### 1. PC 端 — 生成二维码

#### 入口
- 货架导航栏增加「分享」按钮（QR 图标）
- 点击后弹出导出范围选择弹窗
- 选择后根据范围生成对应二维码

#### 数据编码

**单货架格式**：

```javascript
{
  t: 'sf',           // type: shelf_transfer
  v: 1,              // version
  s: {               // shelf
    n: '货架A',      // name
    r: 4             // rowCount
  },
  p: [               // parts
    { c: '1234567890', n: '零件1', s: 'M10', q: 10, r: 0, l: 0, t: '' },
    // c=code, n=name, s=specs, q=quantity, r=shelfRow, l=shelfCol, t=note
  ]
}
```

**多货架格式**（全部导出）：

```javascript
{
  t: 'mf',           // type: multi_shelf_transfer
  v: 1,              // version
  ss: [              // shelves array
    {
      n: '货架A',    // name
      r: 4,          // rowCount
      p: [           // parts
        { c: '1234567890', n: '零件1', s: 'M10', q: 10, r: 0, l: 0, t: '' },
      ]
    },
    {
      n: '货架B',
      r: 3,
      p: [...]
    }
  ]
}
```

#### 编码策略

1. **数据量 ≤ 2500 字节**：直接 JSON 序列化
2. **数据量 > 2500 字节**：使用 pako gzip 压缩，压缩后 Base64 编码
3. **压缩后仍 > 2953 字节**：提示用户「数据量较大，建议减少零件数量」
4. **多货架场景**：全部导出时，若总数据量超过限制，只允许选择部分货架

#### 生成方式

使用 `qrcode-generator` 库（轻量级，无依赖）生成二维码，Canvas 绘制。

#### 自定义 Logo

- 在二维码中心叠加项目图标（复用 PWA 图标 `src/icons/icon-192.png`）
- 白色圆角背景 + Logo，不影响扫码识别

---

### 2. 手机端 — 扫码接收

#### 入口
- 在扫描视图增加「扫一扫导入」按钮（或在顶部工具栏）
- 点击后调用后置摄像头扫描二维码

#### 扫码流程

```
点击「扫一扫导入」
    ↓
启动后置摄像头（复用 Camera 模块）
    ↓
Canvas 截帧 + jsQR 实时识别
    ↓
解析成功 → 判断压缩格式（gz: 前缀则 pako 解压）
    ↓
弹出导入确认弹窗
    ↓
用户选择导入方式 → 导入到本地数据库
```

#### 数据冲突处理（复用 Excel 导入弹窗格式）

| 情况 | 处理方式 |
|------|----------|
| 货架名称不存在 | 直接创建新货架 + 导入零件 |
| 货架名称已存在 | 提示用户选择：覆盖 / 新建货架 / 取消 |

覆盖策略：跳过已占用 / 覆盖 / 遇占用停止（与 Excel 导入一致）

## 新增依赖

| 库 | 用途 | 大小 | 引入方式 |
|----|------|------|----------|
| qrcode-generator | 生成二维码 | ~10KB | CDN `<script>` |
| jsQR | 扫码识别 | ~50KB | CDN `<script>` |
| pako | gzip 压缩/解压 | ~45KB | CDN `<script>` |
