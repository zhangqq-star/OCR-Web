# Excel 导入功能设计 v2.0

## 入口
头部工具栏"导入"按钮（📤），与导出按钮（📥）并列。

## 导入流程

### 1. 选择文件
点击导入 → `<input type="file" accept=".xlsx,.xls">` 选择文件 → SheetJS 读取解析。

### 2. 解析映射
按表头匹配列（兼容导出格式）：

| Excel 列名 | 字段 | 必填 | 新增 |
|-----------|------|------|------|
| 编号 | code | - | |
| 名称 | name | - | |
| 规格 | specs | - | |
| 数量 | quantity | - | |
| 货架行 | shelfRow | - | |
| 货架列 | shelfCol | - | |
| 货架 | shelfName | - | **v2.0 新增** |
| 备注 | note | - | |

- 编号和名称至少有一个，都为空的行跳过
- 货架行/货架列如有则使用（Excel 中 1-based，导入时 -1 转 0-based）
- 无货架行/列的行按顺序自动分配位置
- **v2.0 新增**：支持"货架"列，标识该行属于哪个货架（见下方多货架导入）

### 3. 多 Sheet 选择
- 解析后列出所有 Sheet 名称（如"货架清单"、"Sheet2"等）
- 用户选择要导入的 Sheet，预览随之切换
- 只有一个 Sheet 时不显示选择器

### 4. 多货架导入（同 Sheet 内）
当 Excel 包含"货架"列时：
- 按货架名分组，每组自动创建/匹配同名货架
- 配置弹窗显示各货架的行数分布
- "目标选择"分段控件变为：
  - **按货架名导入**：每行去各自的货架（默认，有"货架"列时）
  - **全部导入到一个货架**：忽略"货架"列，统一导入到选中的货架（覆盖/新建）

无"货架"列时：
- 行为与 v1.0 一致（覆盖现有 / 新建货架）
- **v2.0 新增**：当自动分配的行数超过单个货架容量（4×8=32 格）时，自动溢出到新货架
  - 新货架命名规则：原货架名 + "-续1"、"-续2" ...
  - 溢出时货架行/列归零，按填充方向继续分配
  - 预览网格底部显示"溢出 N 个货架"提示

### 5. 导入配置弹窗
解析完成后弹出配置弹窗，显示：
- **Sheet 选择器**（多 Sheet 时显示）
- **解析结果**：共 N 行数据，M 个货架（有"货架"列时），K 行需自动分配
- **目标选择**（分段控件）
- **填充方向**（分段控件）：先行后列 / 先列后行
- **覆盖策略**（分段控件）：跳过已占用 / 覆盖 / 遇占用停止
- **起始位置选择**：点击预览网格设置起始位置（**v2.0 新增**），默认第一个空格
- **位置预览**：网格显示完整货架，区分已占用/待导入/覆盖/起始点/路径

### 6. 执行导入
确认后按计划写入：
- 有位置信息的行 → 写入指定位置（按覆盖策略处理冲突）
- 无位置信息的行 → 从用户选择的起始位置开始，按方向和策略依次分配
- 有"货架"列的行 → 写入对应的货架（自动创建不存在的货架）
- **溢出处理**：当前货架填满后自动新建续架，行/列归零继续填充，直到所有行分配完毕

### 7. 完成汇总
弹窗显示：导入成功 N 个 / 覆盖 K 个 / 跳过 M 个 / 新建货架 S 个（含溢出创建的续架）

## 技术要点
- SheetJS `wb.SheetNames` 获取所有 Sheet 名
- "货架"列可选，不影响导出格式兼容性
- 起始位置复用连续导入的位置选择交互
- 每个货架独立维护起始位置和占用状态
- 导入模块独立文件 `src/js/import.js`
