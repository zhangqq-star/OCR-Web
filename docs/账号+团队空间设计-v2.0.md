# 账号 + 团队空间 功能设计 v2.0

## 概述

v2.0 将 OCR-WEB 从纯前端本地应用升级为支持在线协同的系统，新增 Node.js 后端服务、用户账号、团队空间和权限管理。

## 核心能力

- **用户账号**：注册/登录/个人信息管理，JWT 认证
- **团队空间**：创建团队、邀请码加入、成员管理(owner/admin/member)
- **三模式运行**：本地模式 / 在线模式 / 离线降级
- **在线协同**：团队成员共享货架数据，实时同步
- **操作日志**：所有变更记录在后端

## 后端设计

### 技术选型

| 组件 | 选择 |
|------|------|
| 框架 | Express |
| 数据库 | SQLite (sql.js) |
| 认证 | JWT + bcryptjs |
| 静态服务 | Express.static (前端同源部署) |

### 数据库表

- `users` — 用户 (username, email 可选, password_hash, display_name)
- `teams` — 团队 (name, owner_id, invite_code)
- `team_members` — 成员关系 (team_id, user_id, role)
- `shelves` — 货架 (team_id, name, created_by)
- `parts` — 零件 (shelf_id, name, code, specs, quantity, note, shelf_row, shelf_col, created_by)
- `operation_logs` — 操作日志 (user_id, team_id, action, target_type, target_id, detail)

### API 端点

- `/api/auth` — 注册、登录、个人信息
- `/api/teams` — 团队 CRUD、加入、成员管理
- `/api/teams/:teamId/shelves` — 团队货架 CRUD
- `/api/teams/:teamId/shelves/:shelfId/parts` — 团队零件 CRUD
- `/api/teams/:teamId/logs` — 操作日志查询
- `/api/personal/shelves` — 个人空间货架
- `/api/personal/shelves/:shelfId/parts` — 个人空间零件

## 前端设计

### 新增模块

- `auth.js` — 登录态管理、token 存储
- `api.js` — HTTP 封装、离线队列、自动同步
- `team.js` — 空间切换、团队管理

### 改造模块

- `db.js` — 新增 spaces/sync_queue 表，v1→v2 迁移
- `app.js` — DataStore 抽象层、账号事件绑定、迁移流程
- `shelf.js` — 空间感知、DataStore 路由
- `export.js` — 空间名加入导出文件名

## 数据流

### 本地模式
```
UI → App (DataStore) → DB (OPFS SQLite)
```

### 在线模式
```
UI → App (DataStore) → API (JWT) → Server DB (SQLite)
                      ↘ DB (本地缓存)
```

### 离线降级
```
UI → App (DataStore) → DB (本地 + sync_queue)
                      → 恢复网络 → API.flushQueue()
```

## 权限模型

| 角色 | 权限 |
|------|------|
| owner | 所有操作 + 删除团队 + 管理成员 |
| admin | 普通操作 + 管理团队信息 + 移除成员 |
| member | 查看/创建/编辑零件和货架 |
