# 账号功能设计 v2.0

## 概述

v2.0 为 OCR-WEB 增加用户账号系统，支持注册/登录，数据通过后端服务持久化。

## 核心能力

- **用户账号**：注册/登录/个人信息管理，JWT 认证
- **双模式运行**：本地模式 / 在线模式
- **数据同步**：登录后数据可存储到服务端

## 后端设计

### 技术选型

| 组件 | 选择 |
|------|------|
| 框架 | Express |
| 数据库 | SQLite (sql.js) |
| 认证 | JWT + bcryptjs |
| 静态服务 | Express.static (前端同源部署) |

### 数据库表

- `users` — 用户 (username, email 可选, password_hash, display_name)
- `teams` — 个人空间 (name, owner_id) — 内部使用
- `team_members` — 成员关系 (team_id, user_id, role) — 内部使用
- `shelves` — 货架 (team_id, name, created_by)
- `parts` — 零件 (shelf_id, name, code, specs, quantity, note, shelf_row, shelf_col, created_by)

### API 端点

- `/api/auth` — 注册、登录、个人信息
- `/api/personal/shelves` — 个人货架 CRUD
- `/api/personal/shelves/:shelfId/parts` — 个人零件 CRUD

## 前端设计

### 新增模块

- `auth.js` — 登录态管理、token 存储
- `api.js` — HTTP 封装

### 改造模块

- `app.js` — 登录/注册 UI、用户状态显示
- `shelf.js` — 通过 DataStore 进行 CRUD
- `export.js` — 导出文件名包含日期

## 数据流

### 本地模式
```
UI → DataStore → DB (OPFS SQLite)
```

### 在线模式
```
UI → DataStore → DB (本地存储)
       ↘ API (JWT) → Server DB (可选同步)
```
