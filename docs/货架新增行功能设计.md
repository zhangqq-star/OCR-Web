# 货架新增行功能设计

## 背景
当前货架固定 4 行 8 列，无法满足不同场景下扩展容量的需求。需要支持每个货架独立配置行数，用户可动态新增/删除行。

## 方案

### 1. 数据层 — 货架表增加 row_count 字段
- shelves 表新增 `row_count INTEGER DEFAULT 4`
- DB schema 版本升至 v5，增加 `migrateV4toV5()` 做 ALTER TABLE
- `DB.createShelf()` 支持传入初始 rowCount，默认 4
- `DB.updateShelfRowCount(id, rowCount)` 新增接口
- `DB.rowToShelf()` 返回 rowCount 字段

### 2. Shelf 模块 — 行数改为按货架实例读取
- 移除 `const ROWS = 4`
- 新增 `getRows()`：读取当前货架的 `row_count || 4`
- `render()`、`openAddForm()`、`openDetail()` 等按当前货架行数渲染
- 连续导入、位置选择、Excel 导入等调用 `Shelf.getRows()` 获取当前货架行数

### 3. UI — 货架导航区增加行数控制
- 在货架名称右侧增加行数显示：`4 行`（点击可展开操作）
- 操作项：「增加一行」、「减少一行」
- 约束：最少 1 行，删除行时若该行存在零件，提示「第 N 行有 X 个零件，删除将清除这些零件，确定吗？」
- 增加行时直接在末尾追加空行，零件不受影响
- 减少行时删除末尾行，超出新行数的零件同步删除

### 4. 连续导入与位置选择
- `renderBatchPositionGrid()`、`openPositionPicker()` 统一使用 `Shelf.getRows()`
- 路径预览和可用位置计算按当前货架实际行数

### 5. Excel 导入
- `Importer` 中渲染预览网格时读取目标货架的 `row_count`
- **不自动扩行**：续架逻辑保留不变，数据超出当前货架容量时仍按现有规则创建续架
- 导入弹窗内增加「增加一行 / 减少一行」操作，让用户在导入前手动调整目标货架容量
- 按货架分组导入时，每个分组目标货架同样可手动调整行数

## 界面调整
- index.html：shelf-nav 内增加行数控制按钮/显示
- style.css：新增行数控件的样式，保持与现有导航按钮风格一致

## 文件变更
- `src/js/db.js` — schema v5 + row_count 支持
- `src/js/shelf.js` — 移除固定 ROWS，按货架实例读取行数，新增行数控制方法
- `src/js/app.js` — 位置选择、连续导入使用 `Shelf.getRows()`
- `src/js/import.js` — 导入预览使用 `Shelf.getRows()`
- `index.html` — 货架导航区增加行数控制 UI
- `src/css/style.css` — 行数控制样式
